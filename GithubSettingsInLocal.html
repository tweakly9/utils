Configure GitHub Private Repo Access

This guide will help you configure access to a private GitHub repository.

Step 1: Generate a Personal Access Token

To access a private repository, you need to generate a personal access token.

Note: Keep your personal access token secure and do not share it with anyone.

Step 2: Clone the Repository

Use the following command to clone the repository:

git clone https://github.com/username/repo.git

Step 3: Configure Git

Set your Git username and email:

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Step 4: Push Changes

After making changes, use the following commands to push them to the repository:

git add .
git commit -m "Your commit message"
git push origin main

Approach 2: Using SSH Keys

If you prefer to use SSH keys instead of a personal access token, follow these steps:

Step 1: Generate SSH Key

Generate a new SSH key if you don't already have one:

ssh-keygen -t rsa -b 4096 -C "your.email@example.com"

Step 2: Add SSH Key to GitHub

Copy the SSH key to your clipboard and add it to your GitHub account:

pbcopy < ~/.ssh/id_rsa.pub

Then, go to GitHub > Settings > SSH and GPG keys > New SSH key, and paste the key.

Step 3: Clone the Repository

Use the SSH URL to clone the repository:

git clone git@github.com:username/repo.git

Step 4: Push Changes

After making changes, use the following commands to push them to the repository:

git add .
git commit -m "Your commit message"
git push origin main

Step 4: Git Extension

If you are using a Git extension, follow these steps to configure it:

Step 1: Install the Extension

Install the Git extension for your IDE or text editor.

Step 2: Configure the Extension

Open the extension settings and configure your GitHub credentials.

Note: Ensure that you use the personal access token or SSH key you generated earlier.

Step 3: Use the Extension

Use the extension to clone repositories, commit changes, and push to GitHub.
